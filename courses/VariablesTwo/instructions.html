Variables Two

Variables are important to master. You can't write a useful program without them.

Let's analyze the syntax (the parts that are important) of variables. Look at this example again:

var is a keyword in JavaScript, you must always spell it exactly like this. It means that you want to create a new variable. The word myname is the name of the variable, we can name it whatever we want.

The equals sign = means that we want to set the contents of the myname variable (left hand side) to "Kate" (right hand side).

We put "Kate" in double quotes, because it has no special meaning to JavaScript, however var (a keyword) and myname (the name of a variable) do have special meaning.

If we said this line in english, we would replace "equal sign" with "is". So this line is pronounced:

The variable myname is Kate.

We follow most lines of JavaScript with a semi-colon ; meaning that we are done with that line of code. This is like a period in english. Remember, each line of code only does one thing.
